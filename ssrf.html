Server Side Request Forgery (SSRF)

A vulnerabilidade SSRF ocorre quando uma aplicação Web permite que o atacante force o servidor a realizar requisições HTTP em seu nome. Isso pode levar ao acesso a recursos internos, escaneamento de portas ou mesmo acesso a serviços protegidos não expostos ao público.

Exemplos notáveis

  • 2019 – Capital One: Uma falha em serviço de metadata AWS permitiu que atacantes coletassem chaves e acessassem dados de clientes.
  • 2021 – Exchange ProxyLogon: Um conjunto de vulnerabilidades explorava solicitações falsificadas para acessar emails internos.
  • 2017 – Uber: Pesquisadores exploraram um endpoint que acessava recursos internos via URL controlada, possibilitando leitura de arquivos locais.

Para mitigar SSRF, é essencial validar e sanitizar todos os URLs fornecidos pelo usuário, restringir saídas do servidor e empregar listas de permissão ao invés de listas de bloqueio.
