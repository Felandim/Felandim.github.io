Insecure Direct Object Reference (IDOR)

O IDOR é uma vulnerabilidade comum em aplicações web. Ela ocorre quando um sistema referencia diretamente um objeto interno (como IDs em URLs) sem checar se o usuário possui permissão para acessá-lo. Como resultado é possível manipular parâmetros e acessar informações de outros usuários.

Geralmente essa falha surge em URLs com IDs sequenciais ou previsíveis. Se um usuário lógico possui acesso a /profile?id=10, nada impede que teste id=11 e visualize dados de outra conta caso não exista validação adequada.

Exemplos notáveis

  • Facebook (2012): Um pesquisador acessou fotos privadas alterando os IDs dos álbuns, explorando um endpoint que não verificava permissões corretamente.
  • Instagram (2019): A plataforma corrigiu uma falha onde é possível visualizar stories privados modificando o ID da requisição de download.
  • India Aadhaar (2018): Um bug em um portal governamental permitiu que informações sensíveis de cidadãos fossem extraídas alterando parâmetros de URL.

Para prevenir IDOR, as aplicações devem validar sempre se o usuário autenticado possui permissão para cada objeto solicitado, além de evitar IDs facilmente enumeráveis.
